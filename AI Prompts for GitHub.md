You don’t need to know everything about GitHub to use it well.  
You just need to know what to ask.

These prompts can help you set up, improve, and grow your repo quickly.

---

### 🛠️ Setup & Structure

Use these to get your repo created and organized properly.

- Can you help me set this project up as a GitHub repo?
- Can you generate a `README.md` for this repo?
- Can you suggest some relevant GitHub **Topics** I can add to this repo?
- Can you create a ZIP containing these files for this repo:  
    `README.md`, `CONTRIBUTING.md`, `SECURITY.md`, `CODE_OF_CONDUCT.md`, `SUPPORT.md`, `LICENSE.md`, `FUNDING.md`?

---

### 🚀 Running & Sharing Your Project

Use these to make your project usable and visible.

- How can I get a **live demo** for this repo up and running with **GitHub Pages**?
- Can you design a **social preview image** for this GitHub repo?
- Can you make a **graphical button link** that takes users directly to my **issue template form**?

---

### 🤝 Collaboration & Community

Use these to make it easier for others to interact with your project.

- What are some good **Issues** I could add to this repo?
- Can you make an **Issue Template** with a form for submissions to this repo?
- Can you make a **Pull Request template** for this repo?
- Can you suggest a few **Discussions** I could add to this repo?

---

### ⚙️ Automation & Windows Tools

Use these to make your project easier to run and distribute.

- Can you write a `.bat` file to run this script automatically on Windows?
- Can you write a `.bat` file that runs several scripts in a specific order when clicked?
- Can you create a one-click `.bat` file that installs all required dependencies?

---

### 📦 Packaging & Distribution

Use these to turn your project into something others can easily use.

- Can we use **Electron** to package this project as a Windows `.exe`?

---

### 🔍 Reviewing & Improving

Use these to refine your work.

- Could you review this **Pull Request** and advise on how to proceed?

---

### ⭐ Growth & Visibility

Use these to get more attention and contributions.

- How can I encourage users to **star, fork, and contribute** to my repo?

---

## How to Use These Prompts

Be specific.

Instead of:

> “Can you generate a README?”

Try:

> “Can you generate a README.md for a **Streamlit** app that analyzes Gmail Takeout data, including setup instructions and screenshots?”

The more context you give, the better the results.
