# Release

## Overview
Describe how releases are prepared and published.

## Before Releasing
- Confirm tests pass
- Confirm docs are updated
- Confirm version numbers are correct
- Confirm changelog is updated

## Release Steps
1. Create or update the release branch if needed
2. Update version numbers
3. Build the project
4. Run final checks
5. Tag the release
6. Publish artifacts
7. Create GitHub release notes

## Post-Release
- Verify deployment
- Verify download/install flow
- Announce the release if applicable
